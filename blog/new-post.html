New Blog Post

Local authoring tool — writes a Jekyll-ready file into blog/_posts/. Not part of the published site.

The filename uses only the first word of the title (cut at the first comma, period, or space), e.g. “My Great Day, Today” → my. The full title is still kept in the post's front matter.

Posting time defaults to “now” in the selected timezone (New York by default) and is written to the front matter with the correct UTC offset for that zone, including New York's daylight saving changes.

Posts are grouped into year-month subfolders (e.g. blog/_posts/2026-06/) — Jekyll reads posts recursively, so this doesn't affect the published URL.

On browsers that support it (Chrome/Edge), clicking “Save Post” opens a save dialog — navigate to the matching blog/_posts/YYYY-MM/ folder (creating it if it's a new month) and the browser will remember that folder until the month changes. On other browsers it downloads the file instead; move it into the right subfolder yourself.
